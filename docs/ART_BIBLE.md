# 貓咪咖啡廳 Art Bible

版本：V0.55.1-alpha｜美術規格與素材清理版

本文件是房間、家具、角色及環境素材的唯一美術規格。邏輯資料仍以 `room-config.js`、`furniture-config.js` 與各角色設定為準；美術不得反向改動 ID、價格、footprint 或存檔座標。

## 1. 視覺定位

- 溫暖、療癒、日系像素咖啡廳；清楚、低噪訊，手機直式縮小後仍可辨識。
- 素材必須為原創或已有明確授權；禁止商業遊戲截圖、開羅官方素材與授權不明素材。
- 場景素材是透明背景像素圖；商店卡片由 UI 產生，不得把白卡、名稱或價格畫進場景素材。

## 2. 唯一等角投影與 Grid

目前 `ROOM_CONFIG` 的真實設定如下：

| 欄位 | 值 |
|---|---:|
| worldWidth | 1560 |
| worldHeight | 1120 |
| cols × rows | 10 × 8 |
| tileWidth | 128 px |
| tileHeight | 64 px |
| 比例 | 2:1 |
| originX | 720 |
| originY | 300 |

投影沿用 `GridSystem`：Grid X 往畫面右下，Grid Y 往畫面左下；兩軸交織為 128×64 菱形。家具 `x/y` 永遠代表 footprint 的邏輯基準格，禁止為手機或桌面建立第二套座標。地板與兩面牆在最上方菱形邊界銜接，牆面素材不得改動 floor origin。

## 3. 世界座標、Anchor 與 Depth

- 直立地面家具：normalized anchor 原則為 `(0.5, 1)`，即腳底中心。
- 地毯與平面裝飾：原則為 `(0.5, 0.5)`，視覺中心對齊 footprint 中心。
- 壁掛物：使用明確牆面接點，`heightClass: wall`，不進入地板 walk blocking。
- 圖片可向上超出 footprint，但不得用透明邊距或隱形碰撞修正位置。
- Depth 依接地 `worldY + layerBias`，不依圖片頂部或固定大數值排序。

## 4. 光源與陰影

- 唯一主光源：左上方窗戶，光線由畫面左上射向右下。
- 左上／上表面為高光面；正面為中間色；右下／內凹面為暗面。
- 落地陰影朝右下，短而硬邊，保持像素階梯；不使用大型寫實 Gaussian blur。
- 家具陰影建議 18%～32% 不透明度；角色橢圓陰影 16%～26%。
- 低家具陰影距離 1～3 px；中型 2～5 px；高家具 4～8 px，均以來源像素計。
- 高光使用暖奶油色，不使用純白大面積；暗面使用暖褐而非純黑。

## 5. 像素密度與輪廓

- Phaser 固定 `pixelArt: true`、`antialias: false`、`roundPixels: true`。
- Canvas 與 CSS 保持最近鄰縮放；禁止瀏覽器平滑插值。
- 1×1 家具建議來源包圍盒：64～128 px 寬、64～160 px 高。
- 2×1 家具：128～224 px 寬、72～176 px 高，主體須真的沿 Grid 軸延伸。
- 2×2 家具／地毯：180～280 px 寬、96～176 px 高。
- 高櫃建議可見高度 112～192 px；底部仍落在同一 anchor。
- 主要外輪廓 2 px（小型細節可 1 px）；內部細節 1 px。
- 統一輪廓色 `#4a3025`，最深暗部可用 `#2d1c16`，禁止大面積純黑。

## 7. 家具比例與高度

- 1×1：視覺寬度約 52～112 世界像素；腳底留在本格，不跨到鄰格中心。
- 2×1：視覺寬度約 108～190 世界像素；不應只比 1×1 大一點。
- 2×2：視覺寬度約 160～240 世界像素；中心對齊 footprint 中心。
- `floor`：地毯、地墊；`low`：貓床、矮桌；`medium`：桌椅、沙發、工作台；`tall`：高櫃、展示櫃、高植物；`wall`：掛畫、窗台家具。
- 椅子低於桌面；桌面高於貓咪背部；高櫃高於桌椅；貓床接近一隻貓的身長；咖啡機不得與整張桌子等大。
- 吧台需保留未來 `staff-work`、`customer-order`、`serve-pickup` 空間。

## 8. 四方向與鏡像

固定方向為 `down-right`、`down-left`、`up-right`、`up-left`；rotation 0～3 依此順序映射。每個方向應有真正的等角素材。缺圖時只能使用 metadata 中已存在的 fallback；不得拼出不存在的 URL。

只有完全左右對稱、無文字、無不對稱抽屜／把手／操作面，且鏡像後光源仍可接受的物件可設 `mirrorAllowed: true`。吧台、收銀台、咖啡機、烤箱、長桌、長椅、展示櫃、工作台、高櫃與沙發均應補真正方向圖。

## 9. 角色基準

- 貓咪：Frame 64×64；目前可見高度約 42～56 px；anchor `(0.5, 1)`。椅面約位於貓咪肩背附近，桌面高於貓咪背部。
- 未來店員：建議 Frame 96×96 或同等像素密度，可見高度約 72～88 px，2.5～3.5 頭身，anchor `(0.5, 1)`；需四方向站立、行走、工作與送餐。
- 未來顧客：與店員接近；坐姿與站姿分開，需四方向。不得以幾何圓形／長方形作正式角色。

## 10. 牆面與環境

正式方向採暖色塗料、木質護牆板、清楚牆腳線與牆角陰影；窗框、窗簾、菜單板、壁燈、掛畫、層架、入口招牌和窗光均須服從同一 2:1 空間與左上光源。牆飾資料預留 `wallSide`、`wallGridPosition`、`wallAnchor`、`heightClass: wall`、`walkBlocking: false`。

## 11. 場景素材與商店縮圖

- 場景素材：透明 PNG、無文字、無卡片底、完整 anchor 與陰影。
- 商店縮圖：由 UI 使用場景 texture 置中於卡片；名稱、價格、持有數只存在 DOM。
- Prototype 可以保留原 SVG 供舊存檔載入，但不得進入正常商店、新遊戲、獎勵或自動生成。

## 12. 素材交付格式

- 正式家具優先透明 PNG；色彩使用 sRGB，整數尺寸，無半透明白邊。
- 檔名沿用現有 furniture ID；重繪不可更名、改 footprint、價格或存檔座標。
- 四方向以明確檔案或 atlas frame 提供，並在 `textureByDirection` 登記。
- 交付須包含：來源尺寸、anchor、footprint、heightClass、光源檢查、方向清單、socket 清單與授權來源。

## 13. Prototype 重繪驗收

移除白卡和素材內文字，改為透明、2:1 等角像素家具；比例符合 footprint，腳底與陰影清楚，至少提供安全四方向策略。營業設備需辨識操作面並保留 interaction socket；地毯不得阻擋行走；重繪只替換 texture/visual metadata，不改 ID、價格、解鎖與存檔。
